# Documentation

## Purpose
User-facing and developer documentation for the DHCP Dashboard project.

## Ownership
Owned by the root AGENTS.md. All documentation must comply with project-wide standards and the DOX framework.

## Local Contracts
- Documentation files are Markdown (`.md`) format.
- API reference must stay in sync with OpenAPI schema generated by FastAPI at `/api/docs`.
- Deployment guide must cover Raspberry Pi 4/5 (ARM64) as the primary target.
- Architecture diagrams must be understandable without external tooling (Mermaid or ASCII).
- All docs must be accessible to Admin, Operator, and Read-Only roles.

## Work Guidance
- `architecture.md` — system architecture diagrams and component descriptions.
- `api-reference.md` — API endpoint documentation (can be auto-generated).
- `deployment.md` — production deployment guide for Raspberry Pi and generic Linux.
- `development.md` — developer setup and contribution guidelines.
- `user-guide.md` — end-user documentation for using the dashboard.
- When implementation changes, update the corresponding doc immediately.

## Verification
- All links in documentation must resolve.
- Deployment guide steps must be reproducible on a fresh Raspberry Pi OS.
- API reference must match the actual endpoints (check via curl or OpenAPI).
- Architecture diagrams must accurately reflect current code structure.

## Child DOX Index
No child DOX files — this folder contains only documentation files, no further nested domains.
